GalaxyLand (Team Project) — My Contribution

본 프로젝트에서 UI 구현 및 Firebase 기반 계정/데이터 연동 기능을 주로 담당했습니다.
빌딩 건설의 핵심 시스템(배치/생성)은 팀원이 구현했고, 저는 해당 시스템을 기반으로 UI 흐름 및 서버 저장/불러오기 로직을 연결하여 실제 플레이 루프가 완성되도록 작업했습니다.

Key Contributions (개발 서브 증빙)

UI 작업 대부분 구현

빌딩 선택/설치 플로우, 로그인 이후 화면 흐름 등 사용자 조작 경로를 UI로 정리 및 구현

Firebase Authentication 기반 로그인/로그아웃 구현

빌딩 건설 데이터 서버 저장/불러오기 연동

건설 로그(배치/선택 등)를 서버에 저장하고, 로그인 시 로드하여 상태 복원

비동기/스레드 이슈 디버깅 및 해결

서버 저장은 정상인데 로드가 되지 않고 디버그 로그도 출력되지 않는 문제를 분석

원인이 메인 스레드가 아닌 곳에서 콜백/처리가 발생하며 Unity 쪽 로직이 정상 실행되지 않는 케이스임을 파악

메인 스레드에서 동작하도록 **Dispatcher(메인 스레드 실행 유도 코드)**를 적용하여 문제 해결

What I Learned

클라이언트(UI/게임 로직)와 서버(Firebase) 연동에서 발생하는 비동기 처리/스레딩 이슈를 실제로 겪고, 재현 → 원인 규명 → 해결까지 경험

기능 구현뿐 아니라 “플레이 흐름이 끊기지 않게” UI/시스템을 연결하는 관점의 작업 방식
